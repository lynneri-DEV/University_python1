### Разбор файла `app.py`   
1. ##### Импорт библиотек  
- `Flask`: Основной класс для создания приложения.
- `render_template`: Позволяет отправлять пользователю HTML-страницы (шаблоны).
- `request`: Используется для получения данных, которые пользователь ввел в форму (например, название задачи).
- `redirect` и `url_for`: Нужны для перенаправления пользователя на другие страницы.
- `app = Flask(__name__)`: Создаем экземпляр приложения.
2. ##### В качестве базы данных создаем словарь
```python
tasks = [
    {"id": 1, "title": "Сдать лабораторную №9", "done": False}
]
```
3. ##### Добавление задачи
- `methods=['POST']:` Маршрут работает только тогда, когда форма отправляет данные (метод POST).
- `request.form.get('title')`: Достаем то, что пользователь написал в поле name="title" в HTML.
- `new_id = ...`: Генерируем новый `ID` (просто прибавляем 1 к длине списка).
- `tasks.append(...)`: Добавляем новую задачу в наш список.
- `redirect(url_for('index'))`: После добавления мы не рисуем новую страницу, а "толкаем" пользователя обратно на главную, чтобы он увидел обновленный список.

4. ##### Отметка о выполнении
- `<int:task_id>`: Это переменная часть пути. Если нажать на галочку у задачи `№2`, ссылка будет `/complete/2`.
- `task['done'] = not task['done']`: Переключатель (инверсия). Если задача была не сделана — помечаем как сделанную, и наоборот.

5. ##### Удаление задачи
`global tasks`: Позволяет изменять список tasks, объявленный в начале файла.
`[t for t in tasks if t['id'] != task_id]`: Создаем новый список, в который копируем все задачи, кроме той, чей `ID` мы хотим удалить.
