## 1. Цель и задачи

**Цель:** Разработать десктопное приложение на языке Python с графическим интерфейсом, реализующее систему регистрации и авторизации пользователя с валидацией вводимых данных и сохранением учётных данных в файл.

**Задачи:**

1. Изучить библиотеку Tkinter для создания графического интерфейса на Python.
2. Реализовать экран регистрации с полями логина и пароля.
3. Реализовать экран входа в систему с проверкой введённых данных.
4. Добавить валидацию логина и пароля с использованием регулярных выражений (модуль `re`).
5. Реализовать сохранение и загрузку учётных данных через файл с использованием модуля `pickle`.
6. Оформить интерфейс приложения в едином цветовом стиле (тёмная тема).

## 3. Используемые библиотеки

### `tkinter` и `tkinter.messagebox`

```python
from tkinter import *
from tkinter import messagebox
```

`tkinter` — стандартная библиотека Python для создания графических интерфейсов. Предоставляет классы и функции для создания окон, кнопок, полей ввода, меток и других элементов интерфейса. Импорт через `*` позволяет использовать все виджеты (Label, Entry, Button и т.д.) без префикса. `messagebox` импортируется отдельно, так как он является подмодулем, и используется для отображения всплывающих диалоговых окон с сообщениями об успехе или ошибке.

### `pickle`

```python
import pickle
```

Стандартный модуль Python для сериализации (преобразования) объектов Python в байтовый формат и их сохранения в файл, а также для обратного считывания. В данном проекте используется для сохранения словаря с логином и паролем пользователя в файл `login.txt` в бинарном виде (`wb`) и последующей загрузки (`rb`). Это позволяет сохранять данные между запусками программы.

### `re`

```python
import re
```

Стандартный модуль для работы с регулярными выражениями (Regular Expressions). Используется для валидации логина и пароля по заданным паттернам. Функция `re.match()` проверяет, соответствует ли строка регулярному выражению, начиная с первого символа.

## 4. Обоснование использования Tkinter

Tkinter был выбран как основной инструмент для создания графического интерфейса по следующим причинам:
- Tkinter входит в стандартную библиотеку Python и не требует дополнительной установки через `pip`. Это делает проект переносимым — он запустится на любой машине с установленным Python без дополнительных зависимостей.
- Предоставляет простой и понятный API для создания оконных приложений. Создание окна, добавление виджетов и назначение обработчиков событий требует минимального количества кода.
- Для реализации двухэкранного приложения с формами ввода Tkinter предоставляет всё необходимое: поля ввода (`Entry`), метки (`Label`), кнопки (`Button`), диалоговые окна (`messagebox`) и главный цикл событий (`mainloop`).
- Приложения на Tkinter работают на Windows, macOS и Linux без изменений в коде.

## 5. Структуры данных

В приложении используются следующие структуры данных:

| Структура | Где используется | Назначение |
|-----------|-----------------|-----------|
| `dict` (словарь) | `save()`, `check()` | Хранение пары логин–пароль: ключ — логин, значение — пароль |
| `str` (строка) | Все переменные цветов, введённые данные | Хранение значений из полей ввода и цветовых констант |
| `bytes` | Файл `login.txt` | Бинарное представление словаря после сериализации через `pickle` |

**Ключевая структура — словарь:**

```python
login_pass_save = {login: password1}
```

Словарь выбран потому что он обеспечивает мгновенный поиск по ключу (`login_input in login_pass_save`) без необходимости перебирать все записи. Это оптимальная структура для хранения пар «ключ–значение», каковыми и являются учётные данные пользователя.

**Цветовые константы:**

```python
BG = "#1e1e2e"
ACCENT = "#7c3aed"
TEXT = "#e2e8f0"
```

Строки с HEX-кодами цветов вынесены в константы, чтобы применять единый стиль ко всем виджетам и легко менять оформление в одном месте.

### 6.2 Очистка экрана

```python
def clear():
    for widget in root.winfo_children():
        widget.destroy()
```

Функция получает список всех дочерних виджетов главного окна через `winfo_children()` и удаляет каждый из них. Это позволяет «переключать» экраны без открытия новых окон — старые виджеты уничтожаются, на их место добавляются новые.

---

### 6.3 Валидация через регулярные выражения

```python
def validate_login(login):
    pattern = r'^[a-zA-Z0-9_]{3,20}$'
    return re.match(pattern, login)

def validate_password(password):
    pattern = r'^(?=.*[A-Za-z])(?=.*\d).{6,}$'
    return re.match(pattern, password)
```

**Паттерн логина** `^[a-zA-Z0-9_]{3,20}$` допускает только латинские буквы, цифры и символ подчёркивания, длиной от 3 до 20 символов. Спецсимволы и пробелы запрещены.

**Паттерн пароля** `^(?=.*[A-Za-z])(?=.*\d).{6,}$` требует минимум 6 символов, при этом обязательно должна присутствовать хотя бы одна буква (`(?=.*[A-Za-z])`) и хотя бы одна цифра (`(?=.*\d)`). Lookahead-операторы `(?=...)` проверяют наличие символов без их «поглощения».

### 6.4 Стилизация виджетов

```python
def style_entry(entry):
    entry.configure(
        bg=ENTRY_BG, fg=TEXT, insertbackground=TEXT,
        relief=FLAT, font=("Arial", 11),
        highlightthickness=1, highlightbackground=ACCENT,
        highlightcolor=ACCENT
    )

def style_button(btn, color=ACCENT):
    btn.configure(
        bg=color, fg=TEXT, relief=FLAT,
        font=("Arial", 11, "bold"),
        cursor="hand2", padx=10, pady=6,
        activebackground=ACCENT_HOVER, activeforeground=TEXT
    )
```

Вспомогательные функции применяют единый стиль к полям ввода и кнопкам. `relief=FLAT` убирает стандартную рамку виджета, `cursor="hand2"` меняет курсор на «руку» при наведении на кнопку, `highlightcolor` задаёт цвет рамки при фокусе на поле ввода.

### 6.5 Регистрация и сохранение данных

```python
def save():
    login = registr_login.get()
    password1 = registr_password1.get()
    password2 = registr_password2.get()

    if not login or not password1 or not password2:
        error_label.config(text="Заполните все поля")
        return

    if not validate_login(login):
        error_label.config(text="Неверный формат логина")
        return

    if not validate_password(password1):
        error_label.config(text="Неверный формат пароля")
        return

    if password1 != password2:
        error_label.config(text="Пароли не совпадают")
        return

    with open("login.txt", "wb") as f:
        pickle.dump({login: password1}, f)

    messagebox.showinfo("Успех", "Регистрация прошла успешно!")
    login_screen()
```

Функция последовательно проверяет: заполненность полей, формат логина, формат пароля и совпадение паролей. При любой ошибке обновляется текст метки `error_label` — ошибка отображается прямо на форме без всплывающего окна. При успехе данные сохраняются в файл через `pickle.dump()` и открывается экран входа.

---

### 6.6 Вход и проверка данных

```python
def check():
    login_input = enter_login.get()
    password_input = enter_password.get()

    if not login_input or not password_input:
        error_label.config(text="Заполните все поля")
        return

    try:
        with open("login.txt", "rb") as f:
            login_pass_save = pickle.load(f)

        if login_input in login_pass_save and login_pass_save[login_input] == password_input:
            messagebox.showinfo("Успех", f"Добро пожаловать, {login_input}!")
        else:
            error_label.config(text="Неверный логин или пароль")

    except FileNotFoundError:
        error_label.config(text="Сначала зарегистрируйтесь")
```

Функция загружает данные из файла через `pickle.load()` и сравнивает введённые логин и пароль с сохранёнными. Блок `try/except` перехватывает ситуацию, когда файл ещё не создан (пользователь не зарегистрирован). При совпадении данных выводится приветственное сообщение с именем пользователя.

## 7. Выводы

В ходе выполнения индивидуальной работы было разработано десктопное приложение на Python с графическим интерфейсом, реализующее систему регистрации и авторизации пользователя. По результатам работы можно сформулировать следующие выводы:

1. **Tkinter** оказался удобным и доступным инструментом для создания простых оконных приложений. Его главное преимущество — это часть стандартной библиотеки Python, что исключает необходимость установки сторонних пакетов.

2. **Модуль `pickle`** эффективно решает задачу сохранения Python-объектов в файл. Однако для продуктивных приложений следует использовать более защищённые способы хранения паролей — например, хеширование через `hashlib` или `bcrypt`.

3. **Регулярные выражения** (`re`) позволяют гибко и лаконично описывать правила валидации входных данных. Паттерны для логина и пароля обеспечивают базовую защиту от некорректного ввода.

4. **Динамическое переключение экранов** через функцию `clear()` — простой и эффективный подход для небольших приложений. Для более крупных проектов лучше использовать архитектуру на основе фреймов (`Frame`) или паттерн MVC.

5. **Вынесение стилей** в отдельные функции (`style_entry`, `style_button`) и цветовые константы позволяет легко изменять внешний вид приложения и обеспечивает согласованность оформления.
